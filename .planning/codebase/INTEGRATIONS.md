# External Integrations

**Analysis Date:** 2026-03-31

- What it's used for: Slash-command interface for skills (chk1 skill)
- Integration method: SKILL.md descriptor file in `~/.claude/skills/`
- Discovery: Automatic via Claude Code's skill discovery
- Communication: Skills are invoked via slash commands; no direct API calls from skills
- Documentation: https://docs.anthropic.com/en/docs/claude-code

**GitHub:**
- Service: Source control hosting
- What it's used for: Repository storage and distribution

- Integration method: `git clone` for installation
- Read access: Public clone via HTTPS

**Anthropic API (Optional):**
- Service: Claude API (future potential)
- What it's used for: Not currently integrated
- Note: Skills run inside Claude Code and use Claude's built-in reasoning

## Desktop Applications & System Integration

**iTerm2 (macOS only):**
- Service: Terminal emulator for macOS
- What it's used for: Tab creation and management in iterm2-tmux tool
- Integration method: AppleScript escape sequences via stdout
- Escape sequences used:
  - Tab title: `\033]0;<title>\007`
  - Tab color (RGB): `\033]6;1;bg;red;brightness;<value>\a` (repeated for green, blue)
  - Background image: `\033]1337;SetBackgroundImageFile=<base64-path>\a`
- Dependencies: Requires iTerm2 installed via Homebrew
- Preferences customization: Script can optionally import iTerm2 plist with optimized settings
  - Location: `skills/iterm2-tmux/iterm2/com.googlecode.iterm2.plist`
  - Settings: Auto-hide tmux client, sync clipboard, startup configuration

**tmux (Terminal Multiplexer):**
- Service: Terminal session multiplexer
- What it's used for: Session creation and management for project directories
- Integration method: Command-line API via `tmux` CLI
- Key commands used:
  - `tmux start-server` - Initialize tmux server
  - `tmux has-session -t <name>` - Check if session exists
  - `tmux new-session -d -s <name>` - Create detached session
  - `tmux send-keys -t <name> <keys> Enter` - Send commands to session
  - `tmux attach -t <name>` - Attach to session
  - `tmux list-sessions` - List all sessions
- Configuration file: `~/.tmux.conf`
- Required settings:
  - `set-option -g set-titles off`
  - `set-option -g allow-rename off`
- Dependency: Installed via Homebrew on macOS

**System Shell (bash/zsh):**
- Service: Shell command interpreter
- What it's used for: Script execution and environment variable sourcing
- Configuration file: `~/.zshrc`, `~/.bashrc`, or equivalent
- Shell profile integration:
  - Auto-start snippet added to shell init for iterm2-tmux
  - Detects iTerm2 context before running
  - Uses `source` to load `~/.config/iterm2-tmux/config`

## Data Storage

**Local Filesystem Only:**
- Configuration storage: `~/.config/iterm2-tmux/config`
- Script installation: `~/.local/bin/` (default) or `$INSTALL_DIR` (configurable)
- Skills installation: `~/.claude/skills/<skill-name>/`
- tmux configuration: `~/.tmux.conf`
- Session backgrounds: `~/.local/bin/.session-backgrounds/` (generated PNG files)
- Lockfiles: Temporary lockfiles for auto-start guard (`/tmp/iterm2-tmux-*.lock`)

**No Database Integration:**
- No SQL databases
- No document databases (MongoDB, DynamoDB, etc.)
- No caches (Redis, Memcached)

**File Storage:**
- Local filesystem only
- Background image generation: Stores PNG files in `~/.local/bin/.session-backgrounds/`
- Generated by: `gen-session-bg.py` using Pillow
- Resolution: 1920x1080 pixels
- Format: RGB PNG with watermark text (session name at opacity 40/255)

## Authentication & Identity

**No Authentication Integration:**
- No OAuth, SAML, or OpenID Connect
- No API key management
- Skills run within Claude Code context (authentication handled by Claude)
- GitHub access: Public repository (no authentication required for cloning)

- Tools are user-local (run under `$HOME` directory)

## Monitoring & Observability

- Not integrated with external error tracking (Sentry, DataDog, etc.)
- Error handling: Local stderr output only

**Logs:**
- No centralized logging

  - Shell environment checks: Detects iTerm2 vs other terminals

## Environment Configuration

**Required Environment Variables:**
- `HOME` - User home directory (used throughout for config/install locations)
- `PATH` - Shell search path (must include tmux, python3, etc.)
- `INSTALL_DIR` - Installation directory override (optional, default: `~/.local/bin`)
- `TMUX_REPOS_DIR` - Projects directory for tmux sessions (optional, default: `~/Repos`)
- `TMUX_SESSIONS_SCRIPT` - Path to tmux-sessions.sh (auto-detected or configurable)

**Optional Environment Variables:**
- `NO_COLOR` - Disables colored output when set
- `REPLY` - Used internally for interactive prompts
- `INTERACTIVE` - Auto-detected based on terminal context (true/false)
- `FORCE` - Skips prompts when set to true
- `ARGUMENTS` - Passed to skills (chk1 uses this for scope specification)

**No Secrets Management:**
- No API keys or tokens required for core functionality
- No .env file dependencies (would be gitignored if present)
- Configuration is stored in plaintext in `~/.config/iterm2-tmux/config`
